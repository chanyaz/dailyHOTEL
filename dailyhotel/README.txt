
** README - ANDROID **

앱 진입점 - MainActivity.java

1. 앱 분석 툴.
   - 화면 트래킹 기능 사용.
   - 이벤트 추적 사용(회원가입, 호텔결제)
     * 회원 가입의 경우 일반 회원 가입만 이벤트 추적이 가능하도록 설계 되어있음. 추후에 페이스북으로 가입을 시도하는 경우도 트래킹 하도록 변경요망
     * 호텔 결제의 경우 계좌이체는 구조상 트래킹이 불가능하며, 카드결제만 트래킹 가능하다.
   - 이커머스 사용(호텔결제)
     * 데일리호텔의 매출액을 산정하기 위하여 호텔 결제를 트래킹함. 하지만 계좌이체를 트래킹하지 못하는 상태여서 부족함.
   - 앱 충돌로 서비스중인 앱 버전의 에러와 버그를 확인하고 수정하도록함.
   
2. 회원가입
   - 페이스북 로그인의 경우 처음 개발자 컴퓨터의 셋팅이 필요함.
     * 페이스북 SDK를 사용하기 위해선 개발자 컴퓨터의 HashKey를 페이스북 개발 콘솔에 등록하여야함(제이슨에게 요청)
       이를 등록하지 않을 경우 페이스북 앱이 설치되어있는 사용자가 페이스북 로그인을 요청할 경우 페이스북 로그인이 불가.
       하지만 페이스북앱이 아닌 웹을 통하여 로그인 할 경우 가능.
     * 페이스북으로 따로 회원가입을 시도하는 메뉴는 존재 하지 않으나, 페이스북으로 로그인을 시도하는 도중, 해당
       페이스북 아이디가 우리 앱에 가입되어 있지 않는 상태라면 회원 가입을 시도하고 로그인을 재시도하도록함.
       
3. 통신
   - 서버에 리퀘스트 콜을 보낼때는 JSESSIONID를 토큰으로 하여 리퀘스트를 요청하기 때문에 이와 관련된 준비를 하고 있어야함.
   - 리퀘스트를 요청 한 후 결과는 String || Json 형으로 반환됨.
   - 서버통신의 주소는 Constant.java에서 모두 관리함.
   
4. 결제모듈
   - 결제 모듈은 현재 KCP 모듈(신용카드)가 적용되어 있으며, 다날 휴대폰 소액결제, 무통장입금등의 결제 방식을 구현하고 있다.
   - 결제 모듈은 웹뷰를 이용하여 모바일 웹으로 진행한다.
   - 웹뷰와 앱의 연동을 위하여 @JavascriptInterface  annotation을 사용한다.
